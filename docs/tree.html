tree quick guide

The tree command

List directory contents as a tree. Below are practical, copy\u2011ready examples. Click the Copy button on any block to put the command on your clipboard.

Installation

Install via your package manager:

Basic usage

Run with no args to show the current directory. Use -a to include dotfiles.

Limit depth

Avoid deep output with -L (levels).

Directories only

Show just folders with -d.

Show sizes & human units

Add -s for sizes in bytes or -h for human\u2011readable.

Match by pattern

Use -P (uppercase) to include only patterns; -I (uppercase i) to exclude.

ASCII vs. Unicode

Force ASCII with -A (helpful for plain logs/CI); default uses box\u2011drawing.

Print file info

Show perms, owner, group, size, and timestamps.

Count only

Summarize totals with -o to save output if needed.

Ignore git repos

Common excludes when working in codebases.

Export to JSON/Markdown (via pipes)

tree doesn\u2019t output JSON, but you can pipe to tools for docs.

Reference

  • -a include hidden \u2022 -d dirs only \u2022 -L level
  • -P include pattern \u2022 -I exclude pattern
  • -h human sizes \u2022 -s print size \u2022 -f full path
  • -A ASCII \u2022 -i no indentation lines
  • --noreport omit summary \u2022 --prune hide empty dirs
